# Provider Rate Comparison

> Side-by-side pricing for MTP, JAP, and DaoSMM across overlapping services.
> Rates are provider cost per 1,000 units (in USD unless noted). Last updated: April 2026.
> Use this when choosing which provider to map a ServiceTier to, and for Phase 3 multi-provider routing decisions.

---

## How to Read This

- **Rate** = what Nitro pays the provider per 1,000 units (not what the user pays)
- **Min/Max** = provider's order quantity limits
- **Refill** = provider offers free re-delivery if count drops
- Lower rate is better for Nitro's margins, but reliability and delivery quality also matter
- **Winner** column shows the cheapest provider, but cheapest ≠ best — factor in delivery speed and completion rate

## Phase 3: Multi-Provider Routing Strategy

When automatic routing is built, use this logic:

### Priority Rules

1. **Nigerian-targeted services** → DaoSMM first (only provider with geo-targeting), no fallback
2. **JAP-exclusive services** (Audiomack, Boomplay, Apple Music, WhatsApp, reviews) → JAP only, no fallback
3. **Everything else** → Route by this priority:
   - **Budget tier** → cheapest provider (use rate comparison above)
   - **Standard tier** → best balance of price and reliability
   - **Premium tier** → highest quality provider regardless of cost

### Failover Logic

```
Try primary provider
  → If error/timeout → try secondary provider
  → If secondary fails → try tertiary
  → If all fail → keep order Pending, alert admin
```

### Cost Optimisation

For overlapping services, the rate difference between MTP and JAP is often small ($0.02–$0.10 per 1K). At scale, this adds up. Track completion rates alongside rates — a provider that's $0.05 cheaper but has 80% completion vs 95% ends up costing more in refunds.
